# 项目状态

## 当前应用能做什么

这个项目现在是一个中文梦境自我探索 MVP 首页雏形，名字是 **析梦 Dream Anatomy**。它的定位是：**你的梦境自我探索工具**。页面可以直接在浏览器里打开，不需要安装任何工具。

它强调梦境记录和温和的荣格式自我探索，不提供诊断、治疗、算命或预测。

目前页面包含：

- 中文首页介绍，说明应用想帮助用户记录梦境，并温柔地探索梦中的内在主题。
- 三个明确入口：快速解析、深度引导、梦境日记。
- 点击入口后，可以切换到对应的基础区域。
- 快速解析区域可以输入梦境碎片，并生成本地 mock 的结构化快速解析结果。
- 快速解析结果包含：梦境整理、核心情绪、主要象征、初步荣格解读、反思问题、温和提醒。
- 快速解析结果会保存到当前浏览器的本地梦境日记。
- 梦境日记区域会显示已保存记录的日期、梦境摘要、主要情绪、主要意象、睡眠质量和分析类型。
- 深度引导区域有 4 个温和问题方向的占位说明。
- 如果没有本地记录，梦境日记区域会显示空状态。

所有内容都在本地浏览器运行，当前不会发送数据到服务器。快速解析结果目前只是本地 mock，梦境记录只保存在当前浏览器的 localStorage 中，不适合替代专业心理支持。

# 下一步可以做什么
适合初学者继续添加的功能：
把本地 mock 快速解析替换为经过后端代理的 OpenAI 或 DeepSeek API 返回结果。
增加梦境详情页，让用户查看某条记录的完整内容。
增加“清空日记”按钮。
给常见梦境象征添加更丰富的中文提示。
增加日期字段，让每篇梦日记有记录时间。
添加简单的深色模式切换。
在 assets/ 中加入图片或图标，让页面更有梦境氛围。
添加基础测试，检查按钮点击后是否显示提示文字。
建议一次只做一个小功能。完成后先在浏览器里打开 src/index.html，确认页面仍然可以正常使用。
