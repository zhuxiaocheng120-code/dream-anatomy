# ACCEPTANCE.md

# 析梦 Dream Anatomy MVP 验收标准

## 1. Overall Goal

本文件用于判断 析梦 Dream Anatomy MVP 是否达到“可以演示、可以继续开发”的状态。

AI 或开发者完成任务后，必须按照本文件逐条说明哪些已完成、哪些未完成、如何测试。

---

## 2. General Acceptance

### A1. 项目可以正常运行

通过标准：

1. 项目可以正常启动
2. 首页可以打开
3. 页面没有阻断性报错
4. 控制台不能出现导致功能不可用的 error

AI 完成后必须说明：

1. 使用了什么启动命令
2. 项目是否成功运行
3. 是否有 warning 或 error

---

### A2. 首页合格

首页必须包含：

1. 产品名：析梦 Dream Anatomy
2. 副标题：你的梦境自我探索工具
3. 简短说明：记录梦境碎片，生成温和的荣格式梦境分析
4. 快速解析入口
5. 深度引导入口
6. 梦境日记入口

不通过情况：

1. 首页为空
2. 按钮无法点击
3. 产品名错误
4. 页面风格像医院系统、算命软件、星座塔罗软件或卡通产品

---

### A3. 快速解析流程合格

用户必须可以完成：

1. 输入梦境碎片
2. 点击“保存并解析”
3. 看到快速解析结果
4. 保存到梦境日记
5. 返回首页或继续深度分析

快速解析结果必须包含：

1. 梦境整理
2. 核心情绪
3. 主要象征
4. 初步荣格解读
5. 反思问题
6. 温和提醒

不通过情况：

1. 没有解析结果
2. 没有保存按钮
3. 输出内容像算命
4. 输出内容像心理诊断
5. AI 编造大量用户没说过的情节
6. 没有提醒“这不是诊断、治疗或预言”

---

### A4. 深度引导流程合格

用户必须可以完成：

1. 输入梦境
2. 系统提出 3–5 个问题
3. 用户回答问题
4. 系统生成 Dream Anatomy Report
5. 保存到梦境日记

问题必须覆盖至少 3 类：

1. 情绪问题
2. 联想问题
3. 现实关联问题
4. 主动性问题
5. 醒后感受问题

不通过情况：

1. 问题超过 5 个
2. 问题太长
3. 问题像审问
4. 问题诱导创伤或过度隐私
5. 直接给用户下心理诊断

---

### A5. Dream Anatomy Report 合格

深度报告必须包含：

1. 梦境摘要
2. 梦境情绪
3. 核心意象
4. 荣格视角
5. 现实生活连接
6. 给你的反思问题
7. 今日小行动
8. 温和提醒

不通过情况：

1. 使用“这个梦一定代表……”这种绝对语气
2. 预测未来
3. 判断吉凶
4. 说用户会发财、倒霉、遇灾、恋爱成功
5. 说用户有抑郁症、焦虑症、PTSD 等心理疾病
6. 提供治疗方案
7. 生成恐吓、宿命论或让用户焦虑的解释

---

### A6. 梦境日记合格

用户必须可以完成：

1. 保存至少一条梦境记录
2. 在梦境日记页看到这条记录
3. 点击记录进入详情页
4. 在详情页看到原始梦境和分析内容

每条记录至少包含：

1. 日期
2. 原始梦境
3. 梦境摘要
4. 情绪
5. 意象
6. 睡眠质量
7. 分析内容

不通过情况：

1. 保存后看不到记录
2. 详情页打不开
3. 记录内容错乱
4. 保存的梦境和显示的梦境不一致

---

### A7. 安全边界合格

整个产品中不能出现：

1. 算命式判断
2. 未来预测
3. 吉凶判断
4. 医疗诊断
5. 心理疾病标签
6. 治疗建议
7. 恐吓用户
8. 绝对化梦境解释

产品中应该出现类似表达：

1. 可能
2. 也许
3. 可以理解为
4. 你可以思考
5. 这不是诊断、治疗或预言，只是一种自我探索视角

---

### A8. UI 风格合格

界面应该符合：

1. 安静
2. 神秘
3. 高级
4. 温和
5. 情绪安全
6. 适合中文用户
7. 适合手机端阅读

不通过情况：

1. 颜色杂乱
2. 图标幼稚
3. 像低级玄学软件
4. 像医院系统
5. 像星座塔罗软件
6. 文字太密
7. 按钮不清楚
8. 随便添加廉价月亮图标

---

## 3. Required Evidence After AI Development

每次 AI 完成开发任务后，必须提交以下说明：

1. 修改了哪些文件
2. 新增了哪些功能
3. 删除了哪些内容
4. 如何运行项目
5. 如何测试每个核心流程
6. 哪些验收标准已经通过
7. 哪些验收标准没有完成
8. 当前已知 bug
9. 下一步建议

---

## 4. MVP Pass Standard

MVP 最低通过标准：

1. 首页可用
2. 快速解析可用
3. 深度引导可用
4. 梦境日记可用
5. 至少能保存并查看一条梦境
6. 所有用户可见文案以中文为主
7. 不出现诊断、治疗、算命、吉凶预测
8. 视觉风格符合 Dream Anatomy
