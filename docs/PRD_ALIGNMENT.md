# PRD 对齐说明

## 1. 当前 MVP 已经完成了什么

**析梦 Dream Anatomy** 当前已经有一个可直接打开的中文静态页面，定位是：**你的梦境自我探索工具**。

已完成内容：

- 首页展示了产品名称、定位和核心说明。
- 页面强调梦境记录和温和的荣格式自我探索。
- 用户可以输入一个梦中象征，并获得一个继续思考的问题。
- 用户可以在梦日记文本框里写下梦境片段。
- 用户可以点击按钮，获得一段轻柔的荣格式反思。
- 项目已有 README、项目状态文档、改进想法文档和基础 Git 提交。

## 2. PRD 里还没有完成的功能

当前版本还只是演示型 MVP，下面这些方向还没有实现：

- 醒来后快速记录梦境的专门入口。
- 引导式梦境分析流程，也就是 3 到 5 个连续问题。
- 梦日记历史，暂时不能保存和回看多篇记录。
- 情绪标签，例如焦虑、平静、困惑、好奇。
- 睡眠质量备注，例如睡得好、醒来疲惫、半夜醒来。
- 象征或主题摘要，例如反复出现的水、房子、考试、迷路。
- 隐私优先的本地保存，目前输入内容刷新后会消失。

## 3. 下一版 v0.2 最应该做的最小功能范围

v0.2 建议只做一个清晰的小范围：**让用户能更顺手地记录一段梦，并在本地保存**。

推荐范围：

- 给梦日记区域改成更明确的“快速记录梦境”。
- 增加日期显示或日期输入。
- 把梦日记保存到浏览器本地存储。
- 页面刷新后自动恢复上一条梦日记。
- 增加一个“清空记录”按钮。
- 保留现有的温柔反思按钮，不增加复杂分析。

这样 v0.2 会更像一个真正能用的小工具，同时仍然适合初学者开发。

## 4. 暂时不要做的功能

为了避免过度开发，下面这些功能先不要做：

- 登录、注册和账号系统。
- 云端同步或数据库。
- 复杂 AI 对话界面。
- 大量心理学理论页面。
- 付费、订阅或会员功能。
- 社区分享、公开梦境广场。
- 复杂图表和长期趋势分析。

这些功能不是不好，而是现在会让项目变重。当前阶段应优先把“记录梦境”做稳。

## 5. 产品边界

产品表达必须始终清楚：

- 不诊断：不判断用户是否有心理疾病。
- 不治疗：不替代心理咨询、心理治疗或医疗建议。
- 不算命：不把梦解释成固定命运。
- 不预测：不声称梦能预测未来。

更合适的表达是：

- “这可能邀请你思考……”
- “你可以温柔地观察……”
- “这个象征也许与你当下的感受有关……”
- “这不是标准答案，只是一个自我探索入口。”

## 6. 下一次编码清单

下次开始写代码前，可以按这个顺序做：

- [ ] 打开 `src/index.html`，确认当前页面能正常显示。
- [ ] 只选择一个 v0.2 小功能，不同时做太多事。
- [ ] 优先实现梦日记本地保存。
- [ ] 确认刷新页面后内容还在。
- [ ] 增加清空按钮，并确认它只清空本地梦日记。
- [ ] 检查页面文案是否仍然强调自我探索。
- [ ] 避免出现诊断、治疗、算命或预测语气。
- [ ] 更新 `README.md` 和 `docs/PROJECT_STATUS.md`。
- [ ] 在浏览器里手动测试一遍。
- [ ] 提交 Git commit。
